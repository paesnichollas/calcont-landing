# Calcont Landing Page

## Analytics (GA4)

1. Set `NEXT_PUBLIC_GA_ID` with your Google Analytics 4 measurement ID (`G-XXXXXXX`) in `.env` or Vercel.
2. When the ID exists, `app/layout.tsx` injects GA scripts after interactive and page views start without blocking render.
3. Track conversion events only through `lib/analytics.ts`:
   - `click_ja_sou_cliente`
   - `click_ainda_nao_sou_cliente`
   - `click_whatsapp`
4. Use `track()` (or `trackEvent()` alias) from `lib/analytics.ts`. If GA is not configured, calls are safe no-op.

## SEO and Open Graph

1. Base SEO config lives in `content/seo.ts`.
2. Global metadata is defined in `app/layout.tsx` using the content config:
   - title template
   - description
   - canonical
   - Open Graph
   - Twitter card
3. Sitemap and robots are generated by:
   - `app/sitemap.ts`
   - `app/robots.ts`
4. Default social preview image is `public/og/og-default.png` and metadata points to `/og/og-default.png`.

## Content Management

1. Copy base is centralized in `content/copy.ts`.
2. Existing section content remains in dedicated files like `content/landing.ts`, `content/services.ts`, and `content/testimonials.ts`.
3. Components must always support empty fields with safe fallback behavior.

## Quick Verification

1. Run `npm run dev`.
2. Open `/robots.txt` and `/sitemap.xml`.
3. Inspect page `<head>` for canonical, OG, and Twitter tags.
4. Click Hero/CTA/Navbar conversion buttons and verify GA events (when `NEXT_PUBLIC_GA_ID` is configured).
